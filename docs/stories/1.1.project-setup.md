# Story 1.1 — Project setup & build pipeline

**Epic:** E1 Foundation · **Status:** Done

## Story
As a developer, I want a Vite project with unit and e2e test runners and two build outputs, so that I can develop fast and ship both a hosted app and a zero-setup file.

## Context (everything the dev agent needs)
- No UI framework — vanilla ES modules. See `docs/architecture.md` §3 for the rationale.
- Tooling chosen in `docs/test-strategy.md`: Vite (dev/build), Vitest + happy-dom (unit), Playwright (e2e).
- The standalone single-file build is a product requirement (PRD success criterion 1): non-technical users double-click `standalone/UkeStart.html`.

## Acceptance criteria
1. `npm run dev` serves the app at :5173.
2. `npm test` runs Vitest against `tests/unit/**` in a happy-dom environment.
3. `npm run test:e2e` runs Playwright and can auto-start the dev server.
4. `npm run build` emits `dist/`.
5. An esbuild step inlines JS + CSS into `standalone/UkeStart.html`, which renders from `file://`.
6. `vitest.config.js` scopes coverage to `src/lib/**` and `src/ui/chordDiagram.js`.

## Implementation notes
- `type: module` in `package.json`; configs are `vite.config.js`, `vitest.config.js`, `playwright.config.js`.
- Keep the dependency list minimal and all dev-only.

## Verification
`npm install && npm test` → all green; `npm run build` → dist emitted; open standalone file → five tabs render.
