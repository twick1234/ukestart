# Story 4.2 — Pure transposition engine

**Epic:** E4 Song Book · **Status:** Done

## Story
As a learner, I want to shift a song's key up or down, so I can sing and play it comfortably without rewriting the chart.

## Context
- Pure module `src/lib/transpose.js`; the UI in `main.js` keeps a per-song semitone offset and re-renders.
- Chord charts store chords as a spaced "lane" string above each lyric (e.g. `"C        F     C"`). Transposing must **preserve column spacing** so chords stay above the right syllables — this is the subtle requirement.
- Use the 12-tone chromatic scale from `src/data/chords.js`. Normalise flats to sharps via an enharmonic map before indexing.

## Acceptance criteria
1. `splitChord('Am')` → `{root:'A', quality:'m'}`; `splitChord('Bb')` → `{root:'A#', quality:''}`.
2. `transposeChord('C', 2)` → `'D'`; wraps the octave (`transposeChord('B',1)` → `'C'`, `transposeChord('C',-1)` → `'B'`).
3. Unknown tokens (e.g. `"(strum)"`) pass through unchanged.
4. `transposeLane(lane, n)` preserves the original string length/spacing.
5. `transposeSong(song, n)` is **immutable** — the input song is not mutated.
6. Covered by `tests/unit/transpose.test.js`.

## Implementation notes
- Lane transposition uses a regex that matches chord tokens (`/[A-G][#b]?[^\s]*/g`) and replaces each in place, leaving whitespace intact. Beware: this means non-chord words starting with A–G could be matched — acceptable in chord lanes, but never run it over lyric text.

## Verification
`npm test` → `transpose.test.js` green (6 tests), including the spacing-preservation and immutability assertions.
