# Story 5.4 — Live microphone tuner

**Epic:** E5 Tuner & Progress · **Status:** Done

## Story
As a learner, I want the app to listen through my microphone and tell me whether each string is flat or sharp, so I can tune accurately without relying on my ear.

## Context
- Keep the layering: the detection algorithm must be a **pure** function in `src/lib/pitch.js` (no Web Audio), so it can be unit-tested with synthetic waveforms. All browser APIs (`getUserMedia`, `AudioContext`, `AnalyserNode`, `requestAnimationFrame`) live in `src/ui/micTuner.js`.
- Reuse the existing pure helpers in `src/lib/tuner.js`: `nearestString`, `centsOff`, `isInTune`, and `noteFromFrequency`.
- The reference-tone tuner must remain as a fallback for users who deny mic access or have no microphone.
- The mic stream must be released when the user leaves the Tuner tab (privacy + battery).

## Acceptance criteria
1. `detectPitch(buffer, sampleRate)` returns a frequency for a clear tone and `null` for silence, sub-threshold signal, or out-of-range pitches.
2. Detection is accurate to within ~10 cents for each open-string frequency (G 392, C 261.63, E 329.63, A 440).
3. The Tuner tab shows a Start/Stop control; while running it displays the nearest string, a cents needle (clamped to ±50), an in-tune (green) / flat (↑) / sharp (↓) status, and the detected Hz + nearest note.
4. Denying permission or having no mic shows a friendly message, not a crash, and the reference tones still work.
5. Leaving the Tuner tab stops the microphone.
6. Pure code is covered by `tests/unit/pitch.test.js`; the tuner UI presence is covered by the Playwright e2e suite.

## Implementation notes
- Algorithm: normalized autocorrelation with RMS gating, edge-trimming, peak pick past the initial downslope, and parabolic interpolation for sub-sample precision. `fftSize` 2048 is a good latency/accuracy balance.
- The animation-frame callback updates DOM nodes by id directly rather than re-rendering the panel, to stay smooth at 60fps.

## Verification
`npm test` → `pitch.test.js` green; open the app, Tuner tab, Start, and confirm a plucked string reads the right name and the needle centres when in tune.
