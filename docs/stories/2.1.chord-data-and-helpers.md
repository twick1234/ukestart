# Story 2.1 — Chord data model & pure helpers

**Epic:** E2 Chord Library · **Status:** Done

## Story
As a learner, I want accurate chord data and the helpers that read it, so the app can show correct diagrams and reason about difficulty.

## Context
- String order is **[G, C, E, A]** (re-entrant tuning). `0` = open, `-1` = muted, `n` = fret. Finger numbers: 1 index, 2 middle, 3 ring, 4 pinky. This convention is load-bearing across diagrams, audio, and tests — do not change it without updating all three.
- Data lives in `src/data/chords.js`; pure helpers in `src/lib/chordLibrary.js` (no DOM — see the layering rule in `CLAUDE.md`).
- Chords carry a `level` (lesson at which introduced) and `family` (major/minor/dominant) for filtering.

## Acceptance criteria
1. `getChord(name)` returns the chord object or `undefined`.
2. `chordsUpToLevel(n)` returns chords with `level <= n`.
3. `chordsByFamily(f)` filters by family.
4. `fingerCount(name)` counts pressed strings; `fretSpan(name)` is max-minus-min fretted fret.
5. `isValidFingering(frets)` rejects wrong length and out-of-range frets.
6. All of the above are covered by `tests/unit/chordLibrary.test.js`.
